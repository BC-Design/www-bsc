2016-05-07, Saturday

don't worry about learning a compiled language like C. (warning: generalizations ahead)

CPUs only understand machine code, and code in any language needs to be converted to machine code to run.  Bash and Python are interpreted, which means you write some code, hit 'enter', and your code is translated into machine code by the BASH or Python interpreter on-the-fly.  This is fine for most tasks, but too slow for, say, most simulations.  Applications that need speed can be written in a compiled language, like C{++} or FORTRAN***, where the program is compiled by a compiler into machine code ahead of time.  It's common practice to use Python as glue—write programs in slower, easier-to-write-and-understand languages like Python that call compiled code written in C where it's needed.  Compiled languages, usually C, are the only option for writing code that runs on embedded microcontrollers.  THAT BEING SAID, you don't need to learn a compiled language to get started with data analysis and visualization.

*** FORTRAN is ancient and can only do a few things, but one of those things is high-speed matrix math
