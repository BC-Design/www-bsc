The open source solution doesn't support private key recovery for multisig group accounts yet, so if you want to get the advantages of security and access despite Coinbase involvement, you'd better know how to combine your keys yourself.

bgillespie
2015-10-20 18:06
So for the information of the channel, the way the Standard Multisig vault works is that a 2 out of 3 private key encryption scheme is generated, you get two out of three private keys, and Coinbase gets two out of three as well.

bgillespie
2015-10-20 18:07
The key you and Coinbase share is password protected, so Coinbase cannot make use of this shared key unless you provide them with the credentials to do so.

bgillespie
2015-10-20 18:07
But because they have one additional private key, you can access funds through coinbase with only one of your private keys, and in particular, using only a password if desired.

bgillespie
2015-10-20 18:08
But because you have access to two private keys, these can be combined (along with the password) to generate the completed private key needed to access the associated bitcoin address.

bgillespie
2015-10-20 18:09
So it provides a certain level of "account recovery" capability without giving Coinbase any unilateral capability.

bgillespie
2015-10-20 18:09
The utility that @brandon linked to is a front-end to the algorithm to combine the two private keys you have access to in order to get into the vault without Coinbase's involvement.

bgillespie
2015-10-20 18:10
Unfortunately: "There is also currently no support for group vaults."

bgillespie
2015-10-20 18:11
So the group vaults system works a bit differently - in this case, Coinbase gets no private keys, and 3 or 5 private keys are distributed to different individuals.

bgillespie

bgillespie
